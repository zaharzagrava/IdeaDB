Class Error1

A few rules.

  1. In a function that is not processError, Error1 can only be thrown with ['input-filtering', 'loud'] types + path that equals to full path of the function Conclusions
  2. So if an error is instanceof Error1 and has a path that = path argument, we can infer that this is our input-filtering error
  3. If an error is instanceof Error1 and has path that !== path argument, we can infer that this is an error that was originated in one of the function's children function and should be wrapped
  4. If an error is not instanceof Error1, then it is an errro that was originated in the function, and so should be wrapped
